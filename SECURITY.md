# Security

## Reporting a vulnerability

Please report security vulnerabilities **privately** so we can address them before they are widely known.

- Contact the maintainers through a private channel (for example, GitHub Security Advisories, or the security contact your team has published internally).
- Do **not** open a public GitHub issue for security bugs; public issues can expose users before a fix is available.

## What to expect

We aim to acknowledge reports within **48 hours** and will work with you on severity, timeline, and disclosure.
